Существующие решения:
1. Simplified Planner for Jira
План создается из фильтра, у него есть даты начала и конца, владельцы. 
Выглядит как кучка прямоугольников-задач, отступы зависят от 
прогресса, вертикальные линии означают какие-то важные даты,
их можно добавлять, редачить, удалять. Есть линия для текущего дня. 
Можно создавать и редактировать задачи, задавать им даты начала и конца, менять оценки, и мышкой перемещать задачи
для изменения их дат начала и конца. 
Задачи сгрупированы по юзеру, у каждого подсчитано,
сколько задач в каком состоянии (норм, просроченные и законченные). Для каждого юзера открывается
таблица с максимальным количеством задач и текущим, и сколько есть ничейных задач. Отслеживается,
если у пользователя нагрузка по количеству задач превысила заданную емкость.
Перемещением задачи можно сменить ее исполнителя.

2. Plangle
Отображение в виде иерархии:
Эпик
	Задача
		Подзадача
		Подзадача
	Задача
Но только два уровня, потому что джира не позволяет создать подзадачу подзадачи.
Есть исполнитель, оценки, ключи, описания	, есть стоимость в долларах для задач.
Есть создание/удаление/редактирование названия элемента для каждой из колонок,
резолв для задач и подзадач.
Есть селектор фильтров. 
Есть селектор типа отображаемых задач: история, баг, улучшение.
Есть перетаскивание для перемещения и, вроде, изменения порядка.
Есть подсчет суммы оценок задач эпика. Есть оценка для стори как сумма оценок подзадач.
Есть суммы оценок по группам (фронт, бэкенд, тестирование, ...). Можно редактировать оценки истории.
Можно быстро создавать подзадачи только задавая оценку им (а остальное задается в настройках
и сохраняется по дефолту).

Можно установить целевую дату, скорость посмотреть какие задачи успеем, какие нет.
Есть прогноз стоимость проекта на какой-то срок (например, двухнедельный спринт)
при разных скоростях.
Есть диаграмма структуры команды (сектор соответствует сумме оценок набора задач с одинаковым
лэйблом, например, backend, qa, ...)
Стоимость задач показывается исходя из оценки и стоимости проекта.
Кнопка создания спринта. Редактирование даты начала и конца спринта. 
Автоматическое заполнение спринта задачами, которые с уверенной скоростью будут выполнены.
Тесная интеграция с Jira Agile.

3. Links Hierarchy for Software & Portfolio
(for Jira Server 7.0.0 - 8.1.0, Jira Data Center 7.0.0 - 8.1.0 and more versions)
Отображение задач и их связей в виде иерархии. Выгрузка в Excel иерархической таблицы задач
со сворачиванием/разворачиванием узлов, ключами, статусом, приоритетом, датой создания и еще чем-то.
Отображение задач со связями в виде графа.

4. Visual Links - view & manage hierarchy
(for Jira Server 7.0.0 - 8.1.0)
Отображение задач в виде иерархии. Отображение в виде графа с редактированием связей на нем.

5. Links Hierarchy for Agile Cloud
(for Agile Cloud)
Отображение задач со связями в виде иерархии.
Можно выполнить JQL-фильтр и посмотреть найденные задачи со связями в иерархии.
From the Issue Navigator, users can export to Excel, build powerful charts, 
make bulk changes and much more. In this way, users can create hierarchical reports for linked issues!

6. Easy Issue Organizer for Jira
(for Jira Cloud and Jira Server 7.1.0 - 8.0.2)
Выводит структуру иерархию Задача < Подзадача, напротив каждой прогресс.
Можно группировать и сортировать и применять джировские фильтры. Можно редактировать названия инлайн.
Можно открыть детали задачи. Можно дублировать и работать в двух вью одновременно.
Есть перетаскивание задач для реорганизации.

7. BigGantt - Gantt Chart for Jira
(for Jira Cloud, Jira Server 7.0.0 - 8.1.0, Jira Data Center 7.0.0 - 8.1.0 and more versions)
Есть таблица задач, но не иерархия, с ключами, названием, прогресс баром с перетаскиванием.
И вью с задачами-прямоуголниками на линии времени и перетаскивание.
Есть экспорт в Excel, pdf, MS Project

8. Issue Archiver for Jira
(for Jira Server 7.2.0 - 7.13.3 and more versions)
Easy to configure settings for archiving and creating snapshot
Secured sharing of archives online with other Jira users with permission access control
Admins can download archived issues (HTML format) and their attachments in zip format to share with remote users
